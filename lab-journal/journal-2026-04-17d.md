## Overflow Analysis

### Before session
One pre-existing overfull after Libertine switch:
- `Overfull \hbox (1.05624pt too wide)` at line 281 — Subsystems table (lXX, 3 columns in narrow two-column layout)

### After float conversion (intermediate state)
Three overflows appeared due to paragraph reflow caused by converting inline tables to floats:
- Line 269: 4.03pt — Subsystems table (lXX with Housekeeping cell)
- Line 380: 7.89pt — paragraph "phase-modulating optic" — word not in hyphenation dict
- Line 434: 3.64pt — paragraph "substrate---no cover glass required" — em-dash prevented line break

### Fixes applied

| Fix | Change | Effect |
|---|---|---|
| Subsystems table | Added `\setlength{\tabcolsep}{4pt}` | Freed ~8pt across 3 columns |
| "phase-modulating" | Added `\hyphenation{phase-mo-du-la-ting}` to preamble | LaTeX can now break at syllable boundaries |
| Em-dash spacing | Changed `substrate---no` to `substrate --- no` | Space around em-dash provides break opportunity |

### After fixes
Zero `Overfull \hbox` warnings. Only remaining warning: pre-existing "Float too large" on the Python code listing figure (was present before this session).
