# Lab Journal — 2026-04-17c — Two-Column PDF Layout Overhaul

## Session Goals
1. Implement two-column layout for `docs/design-and-calculations.tex`
2. Reduce margins to 1in, add column separation and section spacing
3. Fix all table and equation overflows introduced by the narrow column width

## Phase 1: Preamble Changes

Changed `\documentclass` to single-column base; added `\twocolumn` switch after the Table of Contents so the cover page, title page, abstract, and TOC remain full-width (one column), with the main body in two columns starting at Section 1.

| Setting | Before | After |
|---------|--------|-------|
| Document class option | `[10pt,letterpaper]` | `[10pt,letterpaper]` + `\twocolumn` at body start |
| Margin | `1.2in` (geometry) | `1in` |
| Column separator | default 10pt | `\columnsep=14pt` |
| Emergency stretch | none | `\emergencystretch=3em` |
| New packages | — | `tabularx` |
| Section spacing | titlesec default | `\titlespacing*` set (12/6pt for section, 10/4pt for subsection) |

## Phase 2: Table Conversion Strategy

Column width in two-column layout: `(8.5in − 2×1in − 14pt/72) / 2 ≈ 3.18in ≈ 229pt`.

All 19 `\begin{tabular}` environments were converted using two strategies:

**Inline `tabularx`** (15 tables): stays in-column, `X` column absorbs available space.  
**Floating `table*`** (4 original + 8 additional = 12 tables total): spans both columns, gives full 6.5in width.

Final breakdown:

| Environment | Count | Notes |
|-------------|-------|-------|
| `table*` (span both cols) | 12 | mass budget, costs, power budget, schedule, risks, battery density, pointing, visibility, architecture, component lifetimes + 2 tabularx that overflowed |
| Inline `tabularx` | 7 | parameter tables with short `l`-column entries; `XX`, `lX`, `lXX`, `Xrr` specs |
| `\begin{tabular}` unchanged | 0 | all converted |

## Phase 3: Equation Fixes

Several display equations were written for full-page width (6.5in) and overflowed the 3.18in two-column body. Fixed by converting `\[...\]` to `\begin{align*}...\end{align*}` with explicit line breaks at logical midpoints:

| Equation | Overflow (pt) | Fix |
|----------|--------------|-----|
| Ω_arm solid angle | 17.8pt | `align*` split after second `\times` factor |
| P_array align (3rd line) | 17.8pt | `\notag` + continuation line |
| Q_total thermal | 35.9pt | `align*` split before `\approx` |
| d_horizon | 11.1pt | `align*` split before `\approx` |

## Phase 4: Code Listing Fixes

Both `\lstinputlisting` blocks were wrapped in `\begin{figure*}[!ht]...\end{figure*}` to span both columns. Without this, long Python lines (up to 93 chars) overflowed the 229pt column even with `breaklines=true, breakatwhitespace=true`.

## Phase 5: Visualization Figure

Changed `\begin{figure}[h]` → `\begin{figure*}[ht]` for the sky-visualization PNG so it spans both columns.
